             Начнем обучение пожалуй с понимания сетей, ведь считай каждый специалист по ИБ должен в них разбираться.

             IP адрес - уникальный сетевой адрес узла в компьютерной сети. IP адрес есть у каждого устройства, подкоючоного к интернету, а без IP адреса обратиться или обнаружить устройство 
             невозможно.

             DHCP - отвечает за динамическое назначение IP адреса, то есть, когда устройство подключено к локальной сети оно должно открыть IP адрес оно обращается к серверу DHCP который назначает 
             IP адрес системе на фиксированный период времени

             Порт - Если вкратце, то если IP приводит нас к некому устройству, то порт приводит уже к некой службе. Всего портов 65536, но достаточно запомнить следующие:

        20 - FTP
        21 - FTP
        22 - SSH
        25 - SMTP
        53 - DNS
        80 - HTTP
        110 - POP3
        443 - HTTPS

              Протокол - набор правил, определяющий как узлы сети должны передавать данные друг другу.

              TCP - метод, используемый в сети TCP/IP для соединения между локальным клиентом и сервером.
              TCP является двунаправленной связью, а если более простым языокм, стороны прежде чем начать "разговор" должны обменяться приветствиями.

              Итак, если же TCP проверяет отсутствующие пакеты и, если их не находит, запрашивает повторную отправку, то UDP же просто принимает все, что получает, как есть. 
              Это делает UDP быстрее (без дополнительных затрат на проверку, без ожидания потерянных пакетов и т. д.), что делает его лучше для критичных ко времени вещей 
              (прямые трансляции, игры и т. д.)
